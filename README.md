# scooter_project

+ Scooter API autotest

+ Автотест проверяет сценарии:
+
+ - клиент создает заказ через API;
+ - по треку созданного заказа можно получить данные заказа;
+ - код ответа при получении заказа по треку равен 200;
+ - заказ корректно записывается в базе данных.

+ ### Требования
+
+ + Для запуска тестов должны быть установлены пакеты: requests и pytest.
+ +
+ + Перед запуском теста в файле configuration.py необходимо указать:
+ + рабочую ссылку на тестовый стенд (BASE_URL).

+ ### Запуск автотеста
+
+ + В корне проекта выполните команду: pytest -v

+ ### Проверка записи заказа в базе данных
+
+ + Для проверки того, что заказ, созданный через API, отображается в базе данных, выполнялись следующие шаги:
+
+ + 1. Подключение по SSH к учебному серверу (Cygwin64 Terminal):
+ + ssh <имя-пользователя>@<хост> -p 4554
+
+ + 2. Подключение к базе данных scooter_rent:
+ + psql -U morty -d scooter_rent
+ + Пароль пользователя morty: smith.
+
+ + 3. Выполнение SQL-запроса для проверки отображения заказов в статусе «В доставке» (поле inDelivery = true):
+
+ + SELECT c."login",
+ +         COUNT(o."id") AS orders_in_delivery
+ + FROM "Couriers" AS c
+ + JOIN "Orders" AS o ON o."courierId" = c."id"
+ + WHERE o."inDelivery" = true
+ + GROUP BY c."login"
+ + ORDER BY orders_in_delivery DESC;
+
+ + 4. Выполнение SQL-запроса для проверки корректности записи статусов заказов:
+
+ + SELECT
+ +      o."track",
+ +      CASE
+ +          WHEN o."finished" = true THEN 2
+ +          WHEN o."cancelled" = true THEN -1
+ +          WHEN o."inDelivery" = true THEN 1
+ +          ELSE 0
+ +      END AS status
+ + FROM "Orders" AS o;
+
+ + Запрос позволяет убедиться, что значения полей finished, cancelled и inDelivery конвертируются в ожидаемые числовые статусы.

+ ### Наблюдения по результатам проверки
+
+ + При создании заказа через API в базе данных фиксировалось дублирование записи заказа.
+ + По данному поведению ранее заведён баг-репорт.
